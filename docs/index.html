Astriot

Mission

A concept and set of libraries for affordable image processing to extract facial and body pose data (and the like), providing text outputs throw a API for wide range of assistive technologies and other applications.

Problem

Developing assistive technologies is costly because each individual with a disability has unique needs.

For example, in cases of lock-in syndrome (where a patient cannot communicate directly), personalized movement options, such as using the mouth, hands, and similar methods, can be identified. Additionally, the application has other specific requirements, including range of motion capabilities, uncontrolled movements (tics), and more.

Also, hardware components such as a camera and a sufficiently powerful computer for image processing do not have to be cheap.

Solution

Let's divide many specialized tasks into their common component and the remaining parts. For hardware, we will use devices that users already own or can obtain affordably.

Specifically, this means:
  • We'll use united software to capture facial and body pose data, gestures, sounds, etc.
  • This software will run on a smartphone, utilizing its processor for image and sound processing, as well as its camera and/or microphone. It could also potentially leverage other sensors like GPS, accelerometer, gyroscope, light sensors etc.
  • We'll define an API (Application Programming Interface) that other developers can use to create their applications. The API will provide a stream of text data.
  • We'll present several sample applications to demonstrate the full concept of using data from the API.

Safety and ethics

We never transfer video and audio data between applications.

All data is transmitted as general text information only. So, for example, 2D coordinates of the individual joints of the hand or a textual description of the gesture (for example, "open palm" or "fist").

The purpose of the project is to make the entire API (i.e. a detailed description of the data format) public - prepared for user applications. Anyone can check what information is being transferred.

In a nutshell, this means that the same data is transmitted whether you are clothed or not.

If you still don't trust the app, you can test it in a safe environment and after that deny it access to the camera and microphone.

Project status

The project is in the early stages of development. We plan to do the following steps

Supported programming languages

  • JavaScript - for web applications,
  • C++ Arduino like - for embedded devices (with microprocessor)
  • Python
    • - for a wide range of (desktop) applications

Communication API

We want to design an API that will define the format of the transmitted data and commands for communication between the "transmitter" (smartphone) and the "receiver" (application). An abstract description format will be used to generate the appropriate libraries in the above programming languages.

Communication canals

In the programming languages mentioned above, we also want to implement parts of the libraries to support communication between "transmitter" and "receiver" for the following connection methods:

  • WebRTC (PeerJs) for web applications. All modern browsers support the technology.
  • Bluetooth for embedded devices. (We will use the Bluetooth Low Energy (BLE) technology.)
  • Socket for communication with the server (via public WiFi, or in case the application creates a WiFi hotspot)

AstriotTx - Mobile application (transmitter)

  • This is a common application that solves all common problems with image processing (generally sensor data processing).
  • This data broadcasting application is designed as a simple html page that you can open on your mobile. No installation required. Use the link or QR code to open the page on your mobile phone.
  • Further instructions will already depend on the "data receiver" application, most often it will be about selecting a camera (if there are more) and setting it to the observed object. Usually on yourself.

Sample applications (receivers)

  • RxComplexWeb - Test user web application

    In your web browser, you can see what processed data your mobile phone sends from the main application. You can open it in any browser - for example on your desktop.

  • RxBulb - Turn lights on and off using gestures

    A simple, cheap hardware application where we will show you how to easily make a piece of hardware that use visual date from your mobile phone. You can turn the light on and off by opening and closing your palm. We'll use a few trivial components worth about $9 in total.
    We will also show how to modify this application so that you can control the light by blinking.

  • RxLedDisplay - Complex application with LED display

    We will demonstrate different types of use of text data from mobile using a 16x16 RGB LED matrix. We receive and process the data using a cheap ESP32 chip.

  • RxMouseButtons - PowerPoint presentation control with hand gestures

    We will demonstrate the modification of a wireless computer mouse so that its right and left buttons will be controlled by gestures. When connected to any computer, we can effectively move the individual slides of the presentation with the power of gestures.
